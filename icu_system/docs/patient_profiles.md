## Data Characteristics

### Vital Signs Collection
- **Frequency**: Every 15 minutes
- **Duration**: 36 hours of historical data
- **Parameters**: Heart Rate, Systolic/Diastolic BP, SpO2, Respiratory Rate, Temperature

### Temporal Trends
- All patients show **realistic improvement** over the 36-hour monitoring period
- Vital signs gradually stabilize reflecting appropriate ICU management
- Improvement factor: Up to 30% improvement in key metrics over time

### Clinical Realism
- Each condition has **specific vital sign ranges** based on medical literature
- Fever patterns in infectious conditions (sepsis, pneumonia)
- Hypotension in shock states (septic shock, GI bleeding)
- Tachycardia in critical illness
- Hypoxemia in respiratory conditions
- Targeted temperature management in post-arrest care

## API Examples

### Get All Patients
```bash
curl http://localhost:8000/patients/ | python3 -m json.tool
```

### Get Specific Patient Details
```bash
curl http://localhost:8000/patients/2 | python3 -m json.tool
```

### Get Patient Vital Signs (Last 24 hours)
```bash
curl "http://localhost:8000/patients/2/vitals?hours=24" | python3 -m json.tool
```

### Compare Two Patients
```bash
# Septic shock patient (high HR, low BP, fever)
curl -s "http://localhost:8000/patients/2/vitals?hours=1" | tail -1

# Post-cardiac arrest (controlled temp, lower HR)
curl -s "http://localhost:8000/patients/6/vitals?hours=1" | tail -1
```
